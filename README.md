## 📡 Как расшарить Flask-приложение из Docker/WSL в локальной сети (например, для преподавателей и студентов)

Если приложение работает внутри WSL2 и вы хотите, чтобы оно было доступно с других устройств в вашей локальной сети (например, чтобы студенты могли подключиться к вашему демонстрационному приложению), выполните следующие шаги:

---

### ✅ 1. Убедитесь, что Flask-приложение слушает `0.0.0.0`

В файле `run.py` или где запускается сервер, должно быть:

```python
app.run(host='0.0.0.0', port=5000)
```

Если используется `flask run`, добавьте параметры:

```bash
flask run --host=0.0.0.0 --port=5000
```

---

### ✅ 2. Убедитесь, что порт проброшен наружу

Если вы используете `docker run` или `docker-compose`, убедитесь, что порт `5000` проброшен:

```yaml
# docker-compose.yml
services:
  web:
    build: .
    ports:
      - "5000:5000"
```

---

### ✅ 3. Проверьте, что приложение слушает на `0.0.0.0`

Выполните в WSL:

```bash
ss -tuln | grep 5000
```

Ожидаемый результат:

```
tcp   LISTEN 0  4096  0.0.0.0:5000  0.0.0.0:*
```

---

### ✅ 4. Узнайте IP-адрес WSL (внутренний адрес Linux-среды)

```bash
ip addr show eth0
```

Ищите строку вида:

```
inet 172.29.139.69/20 ...
```

→ В данном примере IP WSL: `172.29.139.69`

---

### ✅ 5. Настройка проброса порта в Windows (через PowerShell)

Откройте **PowerShell от имени администратора** и выполните:

```powershell
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=5000 connectaddress=172.29.139.69 connectport=5000
```

---

### ✅ 6. Разрешите входящие подключения через фаервол

```powershell
New-NetFirewallRule -DisplayName "Allow Flask 5000" -Direction Inbound -LocalPort 5000 -Protocol TCP -Action Allow
```

---

### ✅ 7. Узнайте IP хоста Windows

В PowerShell или CMD выполните:

```bash
ipconfig
```

И найдите IPv4-адрес интерфейса Wi-Fi или Ethernet, например: `192.168.1.42`

---

### ✅ 8. Подключение с другого устройства

Теперь с любого устройства в той же Wi-Fi-сети можно открыть:

```
http://192.168.1.42:5000
```

---

### 🔁 Удаление проброса при необходимости

```powershell
netsh interface portproxy delete v4tov4 listenport=5000 listenaddress=0.0.0.0
```

---

### 🧩 Примечания

- Работает только если вы используете WSL2.
- WSL и Windows должны быть на одном хосте.
- Все действия выполняются один раз на запуске — перезагрузка WSL может менять IP.
